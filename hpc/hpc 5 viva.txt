HPC KNN OpenMP Viva Questions & Answers
1. What is KNN?

Answer:
KNN (K-Nearest Neighbors) is a supervised machine learning algorithm used for classification and regression. It classifies a data point based on the majority class of its nearest neighbors.

2. Why is KNN called a lazy learner?

Answer:
Because it does not train a model beforehand. It stores the dataset and performs computation only during prediction time.

3. What is HPC?

Answer:
HPC stands for High Performance Computing. It uses parallel processing and multiple computing resources to solve complex problems faster.

4. How is HPC used in this program?

Answer:
OpenMP is used to parallelize distance calculation and voting operations, improving execution speed.

5. What is OpenMP?

Answer:
OpenMP is an API for parallel programming in C, C++, and Fortran. It allows multiple threads to execute code simultaneously.

6. Why do we include omp.h?

Answer:
omp.h provides OpenMP functions and directives required for parallel execution.

7. What is the purpose of #pragma omp parallel for?

Answer:
It divides loop iterations among multiple threads so they execute in parallel.

8. What are threads?

Answer:
Threads are lightweight execution units that run parts of a program concurrently.

10. Why is Euclidean distance used in KNN?

Answer:
It measures similarity between data points. Smaller distance means higher similarity.

11. What does the Point structure store?

Answer:
It stores x-coordinate, y-coordinate, and class label of each data point.

12. Why is vector used?

Answer:
vector is a dynamic array used to store dataset points efficiently.

13. What is stored in vector<pair<double,int>> dist?

Answer:
It stores distance and corresponding class label.

14. Why are distances sorted?

Answer:
Sorting helps identify the nearest neighbors quickly.

15. What is the role of K in KNN?

Answer:
K represents the number of nearest neighbors considered for classification.

16. What happens if K is very small?

Answer:
The model becomes sensitive to noise and may overfit.

17. What happens if K is very large?

Answer:
The model may underfit and reduce accuracy.

18. Why is K = 3 used here?

Answer:
It provides balanced classification and avoids ties.

19. What is majority voting?

Answer:
The class with maximum neighbor votes becomes the predicted class.

20. What does this line do?
return (count1 > count0) ? 1 : 0;

Answer:
It returns the class with higher vote count using a ternary operator.

21. What is reduction in OpenMP?

Answer:
Reduction combines results from multiple threads safely into a single variable.

22. Why is reduction used in voting?

Answer:
To avoid race conditions while multiple threads update counters simultaneously.

23. What is a race condition?

Answer:
It occurs when multiple threads access and modify shared data at the same time incorrectly.

24. What is supervised learning?

Answer:
It is machine learning where the model learns using labeled training data.

25. Is KNN supervised or unsupervised?

Answer:
KNN is a supervised learning algorithm.

26. What is classification?

Answer:
Classification predicts discrete categories or class labels.

27. What are the classes in this program?

Answer:
Class 0 and Class 1.

28. What is the query point?

Answer:
The query point is the unknown data point whose class must be predicted.

29. What is the query point in this code?

Answer:
The query point is (5,5).

30. Why is output class 1?

Answer:
Because the nearest three neighbors belong to class 1.

31. What is the time complexity of KNN?

Answer:
Time complexity is approximately O(n log n) due to distance calculation and sorting.

32. Which headers are used in this program?

Answer:

iostream → input/output
vector → dynamic arrays
cmath → mathematical functions
algorithm → sorting
omp.h → OpenMP parallelism

33. What does sort(dist.begin(), dist.end()) do?

Answer:
It sorts distances in ascending order.

34. What compiler command is used?

Answer:

g++ -fopenmp knn.cpp -o knn

35. Why is -fopenmp necessary?

Answer:
It enables OpenMP support during compilation.

36. What happens without -fopenmp?

Answer:
Parallel directives are ignored and program runs sequentially.

37. How do you run the program?

Answer:

./knn

Answer:

Answer:

Faster execution
Better CPU utilization
Improved performance
Reduced computation time
